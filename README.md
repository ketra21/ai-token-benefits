## 为什么做这个项目？

当 AI 成为软件开发、研究和创意工作的必备工具，**前沿 AI 模型的访问权限**正在成为一项关键的员工福利——就像免费午餐或健身房会员一样。

本项目追踪哪些公司为员工提供免费的 AI 模型 Token，创建一个透明的、社区驱动的排名体系。它的作用是：

- **求职者**：判断一家公司 AI 化程度的信号
- **在职员工**：对标行业 AI 福利水平
- **公司管理者**：推动对团队 AI 工具的投入
- **行业观察**：从独特视角观察各行业、各地区的 AI 采用情况

## 排名算法

每家公司获得一个 **AI Token 福利分数 (ATBS)**，计算公式：

```
得分 = Σ(模型权重 × 额度倍数 × 访问加成) × 多样性加成
```

| 因素 | 说明 |
|------|------|
| **模型权重** | 前沿模型（Claude Opus, GPT-4.5）权重高于轻量模型 |
| **额度倍数** | 无限制 = 3倍, 高 = 2倍, 中 = 1.5倍, 低 = 1倍 |
| **访问加成** | API+聊天 = 1.5倍, 仅API = 1.3倍, 仅聊天 = 1倍 |
| **多样性加成** | 每多一个 AI 供应商 +10%（多厂商 = 更好） |
|  |  |

**等级体系：**

| 等级 | 分数 | 说明 |
|------|------|------|
| 🏆 S 级 | 120+ | 世界级 AI 福利 |
| 🥇 A 级 | 80-119 | 优秀 AI 福利 |
| 🥈 B 级 | 50-79 | 良好 AI 福利 |
| 🥉 C 级 | 25-49 | 基础 AI 福利 |
| 📋 D 级 | <25 | 最低 AI 福利 |

## 🙋 如何贡献

**你的公司提供免费 AI Token？来告诉我们吧！**

### 方式一：GitHub Issue（最简单）

1. 前往 [Issues](../../issues/new?template=submit_company.yml)
2. 选择「提交公司 AI Token 福利」
3. 填写表单
4. 我们会审核并添加你的数据

### 方式二：Pull Request

1. Fork 本仓库
2. 编辑对应的数据文件：
   - 中国公司 → `data/companies_cn.json`
   - 国际公司 → `data/companies_intl.json`
3. 按照以下格式添加公司条目：

```json
{
  "company": "公司名称",
  "company_en": "English Name",
  "industry": "行业",
  "city": "城市",
  "province": "省份",
  "country": "CN",
  "size": "1000-5000",
  "benefit_mode": "all_employees",
  "models": [
    {
      "provider": "openai",
      "model": "gpt-4o",
      "quota": "high",
      "type": "chat"
    }
  ],
  "verified": false,
  "submitted_by": "你的GitHub用户名",
  "submitted_at": "2026-03-18",
  "source": "employee_report",
  "notes": "补充说明"
}
```

4. 运行验证：`python scripts/validate.py`
5. 提交 Pull Request

所有提交**默认匿名**。

### 字段说明

| 字段 | 必填 | 说明 |
|------|:---:|------|
| company | ✅ | 公司名称 |
| company_en | ❌ | 英文名称 |
| industry | ✅ | 行业 |
| city | ✅ | 城市 |
| province | ❌ | 省份（中国公司） |
| country | ✅ | 国家代码（CN, US 等） |
| size | ❌ | 公司规模 |
| benefit_mode | ✅ | 福利模式：all_employees, tech_only, reimbursement, team_budget |
| models | ✅ | 模型列表 |
| models[].provider | ✅ | 模型供应商（openai, anthropic 等） |
| models[].model | ✅ | 模型 ID |
| models[].quota | ✅ | Token 额度：unlimited, very_high, high, medium, low |
| models[].type | ✅ | 访问类型：api+chat, api, chat |
| verified | ❌ | 由维护者设置 |
| submitted_by | ❌ | 你的 GitHub 用户名（可填 anonymous） |
| notes | ❌ | 补充说明 |

### 额度等级说明

| 额度等级 | 说明 |
|---------|------|
| unlimited | 无限制使用 |
| very_high | 每月 >1000 万 tokens |
| high | 每月 100-1000 万 tokens |
| medium | 每月 10-100 万 tokens |
| low | 每月 <10 万 tokens |

### 福利模式说明

| 模式 | 标识 | 说明 |
|------|:---:|------|
| all_employees | 🟢 全员 | 公司全员统一开放，无需审批 |
| tech_only | 🔵 技术 | 仅技术/研发人员可用 |
| reimbursement | 🟡 报销 | 员工自行购买后报销 |
| team_budget | 🟠 团队 | 由团队/部门预算支出，需主管审批 |

### 提交须知

- **诚实填报**：仅提交你作为在职/离职员工或直接问的某些公司员工，并得到确认的信息
- **可以匿名**：submitted_by 字段可填写 "anonymous"
- **不含机密**：请勿包含任何机密或专有信息
